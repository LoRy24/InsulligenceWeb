# Contributing Guidelines

Thank you for your interest in contributing to this project!
Please follow the guidelines below to help maintain a clear and consistent workflow.

## 1. How to Contribute
### Reporting Issues
- Before opening a new issue, please check if it already exists.
- Provide a clear and descriptive title.
- Include steps to reproduce the problem, expected behavior, and any relevant logs or screenshots.
### Submitting Pull Requests
- Fork the repository and create a dedicated branch for your changes.
- Make sure your code is clear, readable, and consistent with the existing style.
- Keep pull requests focused: one feature or fix per PR.
- Write meaningful commit messages.
- Ensure that all tests pass before submitting the PR.
- Describe the purpose of your changes in the pull request description.

## 2. Coding Standards
- Follow the conventions already used in the codebase.
- Use descriptive names for variables, functions, and classes.
- Keep functions small and focused.
- Add comments when necessary to explain complex logic.
- Remove any unused code or console/debug statements.

## 3. Testing
- Add or update tests for any new features or bug fixes.
- Ensure that the test suite passes before requesting a review.
- If you introduce breaking changes, mention them clearly in the PR description.

## 4. Documentation
- Update the documentation when introducing new features or behaviors.
- Ensure README sections remain clear, concise, and accurate.
- When adding code examples, keep them minimal and easy to understand.
